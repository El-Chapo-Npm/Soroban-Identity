# Issue #339: Dark Mode Support Implementation Progress

**Branch Name:** `feature/dark-mode-system-support`
**Commit Message:** `feat: add system-level dark mode support with toggle - in-progress`

## Issue Summary
The app currently renders only in a light theme. Users who set their OS to dark mode see an eye-straining white UI. The app should honor the `prefers-color-scheme: dark` media query by default and optionally let users override it via a toggle.

## Planned Implementation

### 1. CSS Custom Properties Setup
- Define CSS custom properties for light theme (default)
- Create `@media (prefers-color-scheme: dark)` block with dark theme custom properties
- All color values must use custom properties for easy theme switching

### 2. Theme Management System
- Create theme store/context to manage theme state
- Support three theme modes: 'system', 'light', 'dark'
- Default to 'system' on first visit
- Persist user preference in localStorage under key 'theme'

### 3. Theme Toggle Component
- Add theme toggle button to app header
- Button should cycle: system → light → dark → system
- Use sun/moon icons to indicate current theme
- Update theme immediately on toggle

### 4. Integration Points
- Update `frontend/src/index.css` with CSS custom properties
- Modify `frontend/src/App.tsx` to include theme toggle
- Create or update `frontend/src/store/walletStore.ts` (or separate theme store) for persisted preference

### 5. Testing Requirements
- OS dark mode preference respected on first load with no flicker
- Theme toggle changes active theme immediately
- Selected theme survives page reload
- All UI elements legible in both themes
- Lighthouse accessibility score maintained
